Mozu.Event.Contracts.Event

Properties of an event the system creates each time a create, read, update, or delete operation is performed.

Name Nullable Description Data Type List
auditInfo No

Basic audit info about the item.

 AuditInfo No
catalogId Yes The unique identifier of the catalog associated with the event. int No
correlationId No The unique identifier of the API request associated with the event action, which might contain multiple actions. string No
entityId No The unique identifier of the entity that caused the event. For example, if the event is "product.created", the entity ID value represents the product code of the product that was created. string No
id No The unique identifier of the event. string No
isTest Yes If true, the event record was generated as a test request for an application. bool No
masterCatalogId Yes

The unique identifier of the master catalog associated with the event.

 int No
siteId Yes

The unique identifier of the site associated with the event.

 int No
tenantId Yes

The unique identifier of the tenant associated with the event.

 int No
topic No The type of event that was performed, such as "product.created" or "category.deleted". string No

The application identifer for a subscription.

 string No

id No The unique identifer for a subscription. string No

If true, any events that occur as a result of a direct API call by the same application will not be delivered. This can prevent an endless loop in certain scenarios. bool No

subscribingTenants No This is a tenant that is associated with a subscription. SubscribingTenant Yes
topics No A list of topics that the subscription relates to. string Yes

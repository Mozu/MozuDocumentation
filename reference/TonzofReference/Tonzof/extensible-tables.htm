Mozu.Core.Extensible.Contracts.Attribute

Properties of an attribute used to describe customers or orders.

Name Nullable Description Data Type List
adminName No Name of this extensible attribute defined in Mozu Admin. string No
attributeCode No Merchant-defined code for an extensible attribute. string No
attributeFQN No

The fully qualified name of the attribute, which is a user defined attribute identifier.

 string No
attributeMetadata No List of metadata key-value pairs defined for an extensible attribute. AttributeMetadataItem Yes
auditInfo No

Basic audit info about the item.

 AuditInfo No
content No The localized name and description of the attribute, displayed in the locale code specified for the master catalog. AttributeLocalizedContent No
dataType No The data type of the customer or order attribute, which is Bool, DateTime, Number, or String. string No
displayGroup No Whether the customer or order attribute appears in Mozu Admin only, or in both Mozu Admin and the website storefront. Possible values are Admin and AdminAndStorefront. string No
id Yes

Unique identifier of the attribute.

 int No
inputType No The type of input selection used to define a value for the attribute, which is YesNo, Date, DateTime, List, TextBox, or TextArea. string No
isActive Yes If true, the customer or order attribute is active. bool No
isMultiValued Yes If true, the attribute can have more than one value. bool No
isRequired Yes If true, the attribute must have a defined value. bool No
isVisible Yes If true, the attribute is visible in its defined display group. bool No
namespace No Namespace associated with this extensible attribute. string No
order Yes Integer that represents the sequence of the attribute. int No
validation No Properties used when validating a value entered for an extensible attribute. AttributeValidation No
valueType No An attribute value type is either predefined vocabulary by the admin during attribute set up or it can be "AdminEntered" or "ShopperEntered". The difference between predefined values versus manually entered values is such that the first choice is a set of options to choose from. AdminEntered and ShopperEntered are values that are entered rather than system-supplied and are not stored in the database, but captured during a live commerce operations such as during an order. string No
vocabularyValues No List of valid vocabulary values defined for an attribute. AttributeVocabularyValue Yes

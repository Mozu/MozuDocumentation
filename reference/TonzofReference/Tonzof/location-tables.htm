Mozu.Location.Contracts.FulfillmentType

Properties of a method used to fulfill items for an order. Fulfillment types are system-supplied, and include direct ship (DS) or in-store pickup (SP) at this time.

Name Nullable Description Data Type List
code No The system-supplied code that identifies the fulfillment type, which is "DS" for direct ship or "SP" for in-store pickup. string No
name No The system-supplied name of the fulfillment type, which is "Direct Ship" or "In Store Pickup". string No

Mozu.Location.Contracts.LocationUsage

Configuration properties of a location usage type for a specified site. The direct ship location usage type consists of a single location that represents location that supports direct ship (DS) fulfillment. The in-store pickup location usage type consists of a list of location types that represent locations that support in-store pickup (SP) fulfillment. The store finder location usage type consists of a list of location codes, location types, or both.

Name Nullable Description Data Type List
auditInfo No

Basic audit info about the item.

 AuditInfo No
locationCodes No List of location codes to associate with the location usage. At this time, you can only specify one location code in the request for the direct ship location usage type. string Yes
locationTypeCodes No List of location type codes associated with the location usage. The location service identifies the locations of the designated type. The in-store pickup (SP) and store finder (storeFinder) location usage types allow specification or multiple location type codes. string Yes
locationUsageTypeCode No The system-defined code used to identify the location usage type, which is "DS" for direct ship, "SP" for in-store pickup, or "storeFinder". string No
